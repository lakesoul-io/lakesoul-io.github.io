Flink lakesoul datasource

LakeSoul支持Flink DataSource，使用Flink的DataStream API 和 Table API 执行LakeSoul批式和流式数据读取。

1. 开始使用Flink LakeSoul​

Flink引入LakeSoul依赖的方法：将lakesoul-flink文件夹打包编译后得到lakesoul-flink-2.2.0-flink-1.14-SNAPSHOT.jar，放在flink文件夹下。 为了使用Flink创建lakesoul表，推荐使用Flink SQL Client，支持直接使用Flink SQL命令操作lakesoul表，本文档中Flink SQL格式是在Flink SQL Client界面直接输入命令；Table API 格式需要在代码中编写使用。 切换到flink文件夹下，执行命令开启SQLclient客户端。

# 导入postgreSql元数据库配置信息，启动Flink SQL Client
export lakesoul_home=pg.properties && bin/sql-client.sh embedded -j lakesoul-flink-2.2.0-flink-1.14-SNAPSHOT.jar

1.1 创建catalog​

创建lakesoul类型的catalog，指定catalog类型为lakesoul。指定lakesoul的database，默认为default，即可通过Flink建立lakesoul表。

  1. Flink SQL格式
create catalog lakesoul with('type'='lakesoul');
use catalog lakesoul;
show databases;
use `default`;
show tables;
  1. Table API格式
TableEnvironment createTableEnv = TableEnvironment.create(EnvironmentSettings.inBatchMode());
Catalog lakesoulCatalog = new LakeSoulCatalog();
createTableEnv.registerCatalog("lakeSoul", lakesoulCatalog);
createTableEnv.useCatalog("lakeSoul");
createTableEnv.useDatabase("default");

1.2 建表​

LakeSoul支持通过Flink创建多种类型的表，包括无主键、有hash分区、有range分区以及同时包括range和hash分区。 建表的参数含义

1.分区字段 PARTITIONED BY，用于指定表的range分区字段，如果不存在range分区字段，则省略
2.数据源连接器connector，用于指定数据源类型
3.存储格式format，用于指定表的存储格式
4.存储路径path，用于指定表的存储路径

lakesoul表支持Flink的所有常用数据类型，并与Spark sql数据类型一一对应，使lakesoul表能同时支持Flink和Spark的读写。

  1. Flink SQL格式
-- 创建test_table表，以id和name作为联合主键，以region和date作为两级range分区，catalog是lakesoul，database是default
create table `lakesoul`.`default`.test_table (
            `id` INT,
            name STRING,
            score INT,
            `date` DATE,
            region STRING,
        PRIMARY KEY (`id`,`name`) NOT ENFORCED
        ) PARTITIONED BY (`region`,`date`)
        WITH (
            'connector'='lakeSoul',
            'format'='lakesoul',
            'path'='file:///tmp/lakesoul/flink/sink/test');
  1. Table API格式
// 参考1.1创建catalog
TableEnvironment tEnv = ...
String createUserSql = "create table user_info (" +
        "    order_id INT," +
        "    name STRING PRIMARY KEY NOT ENFORCED," +
        "    score INT" +
        ") WITH (" +
        "    'format'='lakesoul'," +
        "    'path'='/tmp/lakesoul/flink/user' )";
tEnv.executeSql(createUserSql);

1.3 写入数据​

支持Flink按批式和流式写入数据到lakesoul表。写入数据时设定时区，共有两种方法。

# 启动Flink SQLClient时设置
/bin/sql-client.sh embedded -z Asia/Shanghai
-- 在SQLClient命令行界面设置
SET 'table.local-time-zone' = 'Asia/Shanghai';
  1. Flink SQL格式

批式：直接插入数据

insert into `lakesoul`.`default`.test_table values (1,'AAA','98',TO_DATE('2023-05-10'),'China');

流式：构建流式任务，从已存在的表同步数据写入到新的表中

-- 设置 checkpointing 时间间隔
set 'execution.checkpointing.interval' = '2min';

-- 表示将`lakesoul`.`cdcsink`.soure_table表中的全部数据，插入到lakesoul`.`default`.test_table
insert into `lakesoul`.`default`.test_table select * from `lakesoul`.`cdcsink`.soure_table;
  1. Table API格式
// 参考1.1创建catalog
TableEnvironment tEnv = ...
tEnvs.executeSql("INSERT INTO user_info VALUES (1, 'Bob', 90), (2, 'Alice', 80), (3, 'Jack', 75), (3, 'Amy', 95),(5, 'Tom', 75), (4, 'Mike', 70)").await();

1.4 修改/删除表​

  1. Flink SQL格式
ALTER TABLE test_table RENAME TO new_table;
DROP TABLE if exists test_table;
  1. Table API格式
// 参考1.1创建catalog
TableEnvironment tEnv = ...
tEnvs.executeSql("ALTER TABLE test_table RENAME TO new_table")
tEnvs.executeSql("DROP TABLE if exists test_table");

2. Flink查询数据​

支持Flink按批式和流式读取lakesoul表，在Flink SQLClient客户端执行命令，切换流式和批式的执行模式。

-- 按照流式执行Flink任务
SET execution.runtime-mode = streaming;
-- 按照批式执行Flink任务
SET execution.runtime-mode = batch;

使用Flink SQL，指定条件查询的格式 SELECT FROM test_table /+ OPTIONS('key'='value')*/

read-option参数含义

参数含义说明参数填写格式
readtype读类型，可以指定增量读incremental，快照读snapshot，不指定默认全量读'readtype'='incremental'
partitiondesc分区信息，如果未指定分区信息，则默认针对所有分区进行读取'partitiondesc'='region=China'
readstarttime起始读时间戳，如果未指定起始时间戳，则默认从起始版本号开始读取'readstarttime'='2023-05-01 15:15:15'
readendtime结束读时间戳，如果未指定结束时间戳，则默认读取到当前最新版本号'readendtime'='2023-05-01 15:20:15'
timezone时间戳的时区信息，如果不指定时间戳的时区信息，则默认为按本机时区处理'timezone'='Asia/Sahanghai'

2.1 全量读​

支持按批式和流式读取lakesoul表的全量数据，支持distinct、Where、Group by、Having、Join、Order by、Limit等常见的Flink SQL算子。批式是读取当前时间下的最新版本数据。流式是读取当前时间下的最新版本数据，并且一旦存在数据更新，可以自动识别并读取。

  1. Flink SQL格式
-- 设置批式模式，读取test_table表
SET execution.runtime-mode = batch;
SELECT * FROM `lakesoul`.`default`.test_table order by id

-- 设置流式模式，读取test_table表
SET execution.runtime-mode = stream;
SELECT * FROM `lakesoul`.`default`.test_table where id > 3

  1. Table API格式

    执行SQL格式

// 创建批式执行环境
StreamExecutionEnvironment env = StreamExecutionEnvironment.createLocalEnvironmentWithWebUI(new Configuration());
env.setRuntimeMode(RuntimeExecutionMode.BATCH);
StreamTableEnvironment tEnvs = StreamTableEnvironment.create(env);
//设置lakesoul 类型的catalog
Catalog lakesoulCatalog = new LakeSoulCatalog();
tEnvs.registerCatalog("lakeSoul", lakesoulCatalog);
tEnvs.useCatalog("lakeSoul");
tEnvs.useDatabase("default");

tEnvs.executeSql("SELECT * FROM test_table order by id").print();
// 创建流式执行环境
StreamExecutionEnvironment env = StreamExecutionEnvironment.createLocalEnvironmentWithWebUI(new Configuration());
env.setRuntimeMode(RuntimeExecutionMode.STREAMING);
env.enableCheckpointing(2000, CheckpointingMode.EXACTLY_ONCE);
env.getCheckpointConfig().setExternalizedCheckpointCleanup(CheckpointConfig.ExternalizedCheckpointCleanup.RETAIN_ON_CANCELLATION);
StreamTableEnvironment tEnvs = StreamTableEnvironment.create(env);
//设置lakesoul 类型的catalog
Catalog lakesoulCatalog = new LakeSoulCatalog();
tEnvs.registerCatalog("lakeSoul", lakesoulCatalog);
tEnvs.useCatalog("lakeSoul");
tEnvs.useDatabase("default");

tEnvs.executeSql("SELECT * FROM test_table where id > 3").print();
执行API格式
// 参考1.1创建catalog
TableEnvironment tEnv = ...
Table userInfo = createTableEnv.from("user_info");
Table filter = userInfo.filter("order_id=3").select($("name"), $("score"));
filter.execute().print();

+--------------------------------+-------------+
|                           Jack |          75 |
|                            Amy |          95 |

2.2 快照读​

支持Flink对lakesoul表执行快照读取，用户通过指定分区信息和结束时间戳，可以查询结束时间戳之前的所有数据。

  1. Flink SQL格式
-- 对test_table在region=China分区执行快照读，读取的结束时间戳是2023-05-01 15:20:15，时区按Asia/Sahanghai
SELECT * FROM `lakesoul`.`default`.test_table /*+ OPTIONS('readtype'='snapshot','partitiondesc'='region=China','readendtime'='2023-05-01 15:20:15','timezone'='Asia/Sahanghai')*/
  1. Table API格式
StreamTableEnvironment tEnvs = StreamTableEnvironment.create(env);
tEnvs.executeSql("SELECT * FROM `lakesoul`.`default`.test_table /*+ OPTIONS('readtype'='snapshot','partitiondesc'='region=China','readendtime'='2023-05-01 15:20:15','timezone'='Asia/Sahanghai')*/").print();

2.3 增量读​

支持Flink对lakesoul表执行增量读取，用户通过指定分区信息和起始时间戳、结束时间戳，可以查询这一时间范围内的增量数据。 Flink SQL格式

-- 对test_table在region=China分区执行增量读，读取的时间戳范围2023-05-01 15:15:15到2023-05-01 15:20:15，时区按Asia/Sahanghai
SELECT * FROM `lakesoul`.`default`.test_table /*+ OPTIONS('readtype'='incremental','partitiondesc'='region=China','readstarttime'='2023-05-01 15:15:15','readendtime'='2023-05-01 15:20:15','timezone'='Asia/Sahanghai')*/

2.4 流式读​

支持Flink对lakesoul表执行流式读取，流式读基于增量读，用户通过指定起始时间戳和分区信息，可以查询自起始时间戳以后的增量数据。 Flink SQL格式

-- 按照流式读取test_table在region=China分区的数据，读取的起始时间戳是2023-05-01 15:15:15，时区按Asia/Sahanghai
SELECT * FROM test_table /*+ OPTIONS('partitiondesc'='region=China','readstarttime'='2023-05-01 15:15:15','timezone'='Asia/Sahanghai')*/

2.5 cdc读取​

支持Flink对lakesoul表按cdc模式执行读取。需要在创建lakesoul表时将use_cdc设置为true。

-- 创建lakesoul cdc表，op表示操作类型
create table `lakesoul`.`default`.cdc_sink (
            `id` INT PRIMARY KEY NOT ENFORCED,
            name STRING,
            score INT,
            op STRING
        ) WITH (
            'connector'='lakeSoul',
            'format'='lakesoul',
            'use_cdc'='true',
            'path'='file:///tmp/lakesoul/flink/sink/cdc_test');

从源表cdc_info中同步数据，在cdc_sink表中可以保留每条记录的操作信息。

insert into `lakesoul`.`default`.cdc_sink select * from `lakesoul`.`cdcsink`.cdc_info;
